This document serves as an instruction, when the setup is already built and in need to connect to the software
For any component information / which exact optical components I used, you can scan through my thesis

connecting the Hardware
1. go to thorcam software and look at the image: is there interference? is the interference in the middle? Because the hardware ROI is set to exactly 40x40 pixels in the middle
2. for the stage
    2.1 pi stage
        2.1.1 go to pimikromove software and select the stage then go to axes and refposition and start to initialize the stage put a check on servo
        2.1.2 in handler_stage.py change self.min_position and self.max_position to 13 each (this stage has 260mm travel range)
    2.2 Thorlabs Kinesis Stage (e.g. LTS150 / LTS300)
        2.2.1 Serial Number Setup:
              - The code uses a default serial number "45517804" (configured in thor_handler_stage.py).
              - If your controller has a different serial number, you must change it in thor_handler_stage.py inside the __init__ method: self.serial_no = "YOUR_SERIAL_NUMBER".
              - You can find the serial number on the label on the physical controller box or in the Kinesis software.
        2.2.2 Travel Limits:
              - In thor_handler_stage.py, change self.min_position and self.max_position. By default, it is configured for 300mm travel range (0.0 to 300.0)
        2.2.3 CRITICAL - Connection Lock:
              - Open Kinesis to verify that the stage is working/homed if needed.
              - IMPORTANT: You MUST close the Thorlabs Kinesis software (and any Simulator) completely before running the Python script. If Kinesis is open, it locks the USB port and the Python script will fail to connect.
3. close both Kinesis and ThorCam apps and then run the program


Starting the software
0. Important: ThorCam, Thorlabs Kinesis, and PIMikromove are Hardware-Drivers/Control-Software. They need to be installed on the computer to communicate with the hardware.
1. install git for windows and then go to powershell and try git --version
    1.5 you need git for sipyco
2. install a python version that is newer than 3.10 and try python --version
3. same with anaconda3
4. install everything else using conda
    4.1. in vs code do ctr shift P --> Python:select interpreter --> choose conda with python.exe
    4.2. install pip and then pip install -r requirements.txt
        5.5 if sipyco from git doesnt work try: python -m pip install git+https://github.com/m-labs/sipyco
    6. install local thorlabs sdk cd "C:\Users\(username)\Desktop\translationstagecalibrationcode\thorlabs_tsi_sdk-0.0.8" C:\ProgramData\anaconda3\python -m pip install .
    7. go to https://www.thorlabs.com/software-pages/thorcam/ and in programming interfaces install thorlabs sdk for windows
    8. For Thorlabs stages, install the official Thorlabs Kinesis software (64-bit). The DLLs will be loaded from the default path `C:\Program Files\Thorlabs\Kinesis\`. Also ensure 'pythonnet' is installed (listed in requirements.txt) to interface with the .NET DLLs.
    9. start the program :)
5. OR install everything else using a venv
    5.1 install a new venv
        5.1.1 using macOs
            python3 -m venv .venv314
            source .venv314/bin/activate
            python -m pip install -r requirements.txt
            then e.g. python main_camera.py
        5.1.2 using windows powershell
            py -3.14 -m venv .venv314
            .\.venv314\Scripts\Activate.ps1
            python -m pip install -r requirements.txt
            then e.g. python main_camera.py




if using photodiodes/front-back measurement
1. download national instrument software https://www.ni.com/de/support/downloads/drivers/download.ni-daq-mx.html#590033
2. python -m pip install nidaqmx or when still in conda environment  C:/Users/FIDOPA_3/anaconda3/python.exe -m pip install nidaqmx
3. make sure, the beam is properly coupled into the photodiode (you can look at the "test device" window, that opens when you plug in the National Instruments usb device, that connects to the photodiodes)
    3.1 check, if there is the equal amount of signal left blocking the beams one after another


if using the camera software and not all fringes get detected
0. restart the software
1. play with the velocity and see if it gets better
2. use longer travel distances and see if it gets better
3. adjust the values from which a fringe gets detected as dark or bright
    #these values I found by experimenting with the setup work best for accurate fringe detection, but could need adjusting
    self.dark_threshold = (
        min_val
        + value_range * 0.125 #<-- change this value if needed
    )

    self.bright_threshold = (
        max_val
        - value_range * 0.60 #<-- change this value if needed
    )
4. Don't touch the optical table!
